## 🔍 审阅维度

### 1. 代码质量 (Code Quality)
- **代码规范**: 命名约定、缩进、注释质量
- **代码复杂度**: 函数长度、圈复杂度、嵌套深度
- **代码重复**: DRY 原则遵守情况
- **可读性**: 变量命名、逻辑清晰度
- **模块化**: 职责分离、耦合度

### 2. 架构设计 (Architecture)
- **模块划分**: 核心模块职责是否清晰
- **进程通信**: Electron IPC 使用是否合理
- **WebSocket 架构**: 连接管理、重连机制
- **状态管理**: 应用状态同步策略
- **扩展性**: 新功能添加的便利性

### 3. 安全性 (Security)
- **API Key 管理**: 加密存储实现（safeStorage）
- **输入验证**: 用户输入、配置文件验证
- **进程隔离**: preload 脚本安全沙箱
- **依赖安全**: 第三方库漏洞检查
- **敏感信息**: 日志中是否泄露密钥

### 4. 性能 (Performance)
- **启动速度**: 主进程加载时间
- **内存使用**: 内存泄漏风险
- **WebSocket 效率**: 消息处理性能
- **缓存策略**: cache-manager 实现
- **资源占用**: CPU、磁盘 I/O

### 5. 错误处理 (Error Handling)
- **异常捕获**: try-catch 覆盖率
- **错误恢复**: 自动重启、回滚机制
- **日志记录**: 错误日志完整性
- **用户反馈**: 错误提示友好性
- **边界情况**: 网络断开、文件缺失等

### 6. 依赖管理 (Dependencies)
- **依赖版本**: 是否使用最新稳定版
- **依赖数量**: 是否存在冗余依赖
- **安全漏洞**: npm audit 检查
- **许可证合规**: 依赖许可证兼容性

### 7. 文档完整性 (Documentation)
- **README**: 安装、配置、使用说明
- **代码注释**: 关键逻辑注释
- **API 文档**: 模块接口说明
- **变更日志**: CHANGELOG 更新

### 8. 测试覆盖 (Testing)
- **单元测试**: 核心模块测试
- **集成测试**: 模块间交互测试
- **手动测试**: 功能验证清单

## 📊 审阅输出

### 审阅报告结构
1. **执行摘要** - 整体评估和关键发现
2. **详细发现** - 按模块分类的问题清单
3. **风险评级** - 高/中/低风险分类
4. **改进建议** - 具体可执行的优化方案
5. **优先级排序** - 修复优先级建议

### 问题分级标准
- **🔴 Critical** - 安全漏洞、数据丢失风险、崩溃问题
- **🟠 High** - 性能瓶颈、内存泄漏、重要功能缺陷
- **🟡 Medium** - 代码质量问题、可维护性问题
- **🟢 Low** - 代码规范、注释完善、文档优化
